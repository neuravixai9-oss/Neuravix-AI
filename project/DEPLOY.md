# Деплой Neuravix AI на Railway

## 1. Создай проект

1. Зайди на [railway.app](https://railway.app), New Project → **Deploy from GitHub repo**
   (залей этот код в свой GitHub-репозиторий) — либо **Empty Project** и подключи
   репозиторий позже.
2. Railway сам определит, что это Python-проект (по `requirements.txt` и
   `.python-version`), установит зависимости и запустит `python main.py`
   (команда задана в `railway.json` / `Procfile`).

## 2. Переменные окружения

В настройках сервиса → **Variables** добавь:

| Переменная | Обязательна | Где взять |
|---|---|---|
| `BOT_TOKEN` | ✅ да | у [@BotFather](https://t.me/BotFather) |
| `GEMINI_API_KEY` | ⚠️ желательно | [Google AI Studio](https://aistudio.google.com/apikey) |
| `OWNER_ID` | ✅ да | твой числовой Telegram ID, узнать у [@userinfobot](https://t.me/userinfobot) |
| `OWNER_USERNAME` | нет | твой username без `@` |
| `SUPPORT_USERNAME` | нет | username поддержки без `@` |

Без `BOT_TOKEN` и `OWNER_ID` бот не запустится — при старте он сам напишет
в логи, какой именно переменной не хватает. Без `GEMINI_API_KEY` бот
запустится, но нейросеть, изображения и перевод работать не будут (в логах
будет предупреждение, а не ошибка).

## 3. Хранение базы данных (важно!)

Бот использует SQLite-файл. Файловая система Railway **не сохраняется между
деплоями** без подключённого Volume — при каждом передеплое база обнулится.

Чтобы данные (пользователи, диалоги, подписки) сохранялись:

1. В сервисе → **Volumes** → создай Volume, примонтируй, например, в `/data`.
2. Добавь переменную окружения `DB_PATH=/data/neuravix.db`
   (или ничего не добавляй — бот сам возьмёт путь из `RAILWAY_VOLUME_MOUNT_PATH`,
   если Volume примонтирован).

Без Volume бот тоже прекрасно работает — просто данные будут сбрасываться
при каждом новом деплое кода.

`DATABASE_URL` поддерживается как альтернативный способ задать путь к тому же
SQLite-файлу (например `sqlite:////data/neuravix.db`) — полноценный Postgres/MySQL
текущая версия бота **не поддерживает**, вся логика написана под SQLite.

## 4. Проверка

После деплоя открой **Deploy Logs** — при успешном запуске увидишь:

```
✅ База данных инициализирована
🤖 Бот: @имя_бота (ID: ...)
✅ Neuravix AI готов к работе!
```

Если видишь `❌ Бот не может запуститься` — в логах прямо под этой строкой
будет список переменных, которых не хватает.

## 5. Важно про тип сервиса

Это Telegram-бот на long polling (не веб-сервер) — ему **не нужен** публичный
домен и открытый порт. Не подключай Public Networking для этого сервиса.
